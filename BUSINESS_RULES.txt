DC FLIGHT DEAL ALERTER — BUSINESS RULES
========================================
Last updated: March 2026 (Destinations reduced to 21; IAD only; 46-min rolling scan)


ORIGINS
-------
1. Only flights departing from the Washington DC metro area are tracked.
2. The origin airport is: IAD (Dulles). DCA and BWI are excluded — DCA serves mostly
   domestic short-hops; BWI is Spirit/Southwest-heavy (both excluded airlines).
3. IAD is searched on every job run.
4. (Reserved — DCA fallback no longer applicable.)


DESTINATIONS
------------
5. 21 destinations are monitored, grouped into six tiers:
     - Domestic Tier 1 (5):   LAX, MIA, BOS, ORD, LAS
     - Caribbean & Mexico (3): CUN, MBJ, PUJ
     - Europe Tier 1 (4):     LHR, CDG, FCO, BCN
     - Europe Tier 2 (2):     LIS, MAD
     - Middle East (4):       BEY, TLV, IST, CAI
     - Asia-Pacific (3):      NRT, BKK, SYD
6. Destinations are scanned in priority order — Domestic Tier 1 is always scanned first,
   ensuring high-demand routes are never skipped if the monthly budget runs out.
7. Scans run continuously every 46 minutes, processing 1 destination × 1 date combo per run.
   A full sweep of all 72 slots completes in ~55 hours.
   (16 non-weekend dests × 2 windows = 32 slots; MIA, BOS, CUN, MBJ, PUJ × 8 combos each = 40 slots)


FLIGHT FILTERS
--------------
8. Spirit Airlines flights are excluded from all price tracking and deal alerts.
9. Frontier Airlines flights are excluded from all price tracking and deal alerts.
10. Flights with 2 or more layovers (2+ stops) are excluded from all tracking and alerts.
11. For domestic destinations (Domestic Tier 1), only nonstop flights (0 stops) are
    eligible for deal alerts. If no nonstop options are returned, the route is skipped.
12. For international destinations, nonstop flights are preferred but 1-stop flights
    are also eligible for deal alerts.
13. For east coast destinations (MIA, BOS) and Caribbean/Mexico (CUN, MBJ, PUJ),
    deal alerts are only sent for weekend travel: departure must be on a Thursday or Friday,
    and return must be on a Sunday or Monday. Non-weekend date combinations are silently skipped.
14. Caribbean/Mexico (CUN, MBJ, PUJ) use the same 4-combo weekend search as east coast
    domestic (MIA, BOS): all valid Thu/Fri × Sun/Mon combos for both near-term and far-out
    booking windows. Trip lengths: 2–4 nights (Fri→Sun, Thu→Sun, Fri→Mon, Thu→Mon).


DEAL DETECTION
--------------
15. A route must have a minimum of 10 price observations before deal detection is
    activated. Routes with fewer observations are skipped (cold-start guard).
16. A flight qualifies as a deal only when BOTH of the following conditions are met:
      a. Current price is at or below 40% less than the historical average
         (current_price <= mean × 0.60)
      b. Current price is below the statistical lower bound
         (current_price < mean − 1.5 × standard deviation)
17. Historical statistics are computed using prices observed within a ±14-day window
    around the departure date to account for seasonal variation.
18. Only price observations from the past 90 days are included in the historical average.
19. The best available price per search result is defined as the lowest price with the
    fewest stops (nonstop preferred). For domestic routes, only nonstop prices are
    considered (see Rule 11).


ALERT RULES
-----------
20. Once a deal alert is sent for a specific route and departure date, no further alerts
    will be sent for that same route and departure date until the cooldown expires:
      - Domestic & Caribbean/Mexico: 3-day cooldown (short booking window)
      - Europe, Middle East, Asia-Pacific: 7-day cooldown
21. Deal alerts are sent via Gmail to the configured ALERT_RECIPIENT email address.
22. Each alert email includes: origin, destination, alert price, historical average,
    savings percentage, dollar savings, airline, stops, travel dates, and a direct link
    to search the deal on Google Flights.
23. An optional weekly digest email summarizing API usage, deals found, and recent alerts
    can be enabled via the WEEKLY_DIGEST setting (disabled by default).


SEARCH SCHEDULE & API BUDGET
-----------------------------
24. The scheduler runs a search job every 46 minutes, processing 1 destination × 1 date
    combo per run using a persistent sweep_cursor (slots 0–71) that rotates continuously.
    Slot order: each destination's full set of date combos in SEARCH_PRIORITY order.
25. Each run uses 1 API credit. Full cycle: 72 slots × 46 min = 55.2 hours.
    Usage: ~31.3 credits/day → ~939 credits/month ✓
26. The monthly API credit budget is capped at 950 credits (SerpApi Starter plan provides
    1,000 credits/month; 50 credits are held as a buffer).
27. If the monthly budget ceiling is reached, all remaining job runs that month are skipped.
28. A warning is logged when API usage reaches 80% of the monthly budget.
29. Date windows are split by destination type. Each slot uses one date combo per run,
    so every destination cycles through all its combos within each ~55-hour sweep:
    Weekend Destinations (MIA, BOS, CUN, MBJ, PUJ) — 8 slots each, all valid combos:
      - Near-term (~2 weeks out): Thu→Sun (3n), Thu→Mon (4n), Fri→Sun (2n), Fri→Mon (3n)
      - Far-out   (~6 weeks out): Thu→Sun (3n), Thu→Mon (4n), Fri→Sun (2n), Fri→Mon (3n)
    Domestic Other (LAX, ORD, LAS):
      - Near-term: ~2 weeks out, 4-night stay  (long weekend)
      - Far-out:   ~6 weeks out, 4-night stay  (long weekend)
    Europe (LHR, CDG, FCO, BCN, LIS, MAD):
      - Near-term: ~3.5 months out, 7-night stay
      - Far-out:   ~7 months out, 7-night stay
    Middle East & Asia-Pacific (BEY, TLV, IST, CAI, NRT, BKK, SYD):
      - Near-term: ~3.5 months out, 10-night stay
      - Far-out:   ~7 months out, 14-night stay
    Departures snap to the nearest target weekday from today + offset_weeks.
30. A one-time seed run (21 dests × 1 origin × 2 windows = 42 credits minimum) is required
    at first launch to bootstrap the price history database before deal detection can activate.


API RELIABILITY
---------------
31. Each SerpApi call is retried up to 3 times on failure, using exponential backoff
    starting at 10 seconds with up to 5 seconds of random jitter.
32. A polite delay of 1.5 to 3.0 seconds is applied between consecutive API calls
    to avoid rate limiting.


DATA RETENTION
--------------
33. All observed prices are stored in a local SQLite database (flights.db).
34. Historical statistics use only prices from the past 90 days.
35. The database is never overwritten by code deployments or updates.


DASHBOARD
---------
36. The web dashboard displays live API credit usage pulled directly from the SerpApi
    account endpoint, refreshed every 5 minutes. If the SerpApi endpoint is unreachable,
    the locally tracked credit count is shown as a fallback estimate.
37. The dashboard is accessible at http://localhost:5000 when running locally, or at
    the server's static IP when deployed to AWS Lightsail.
